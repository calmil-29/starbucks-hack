### 1/23 - 1/25: Home画面の再定義（設計フェーズ完了）
> **「迷い」をゼロにするHome画面の再構築**
> アプリ（Web）を開いた瞬間の「今すぐ支払いたい」「今すぐ注文したい」という2大ニーズに対し、0タップで応えるインターフェースを設計。

#### Headerの完全廃止と「Thumb Zone（親指圏内）」への集約
画面上部のナビゲーションをあえて排除し、すべての操作を指が届きやすい画面下半分に集約。ハンバーガーメニューすら下部に配置することで、片手操作の利便性を極限まで高めています。

####  「0タップ・アクセス」による決済体験の高速化
Home画面のHeroエリアに「カード残高」「QRコード」「スター数」を常駐。**PayPay等の決済特化型モデル**を参考に、「アプリ起動 ＝ 支払い準備完了」という、決済速度の圧倒的向上を狙いました。

#### メンタルモデルに基づいた機能の「統合」と「分離」
* **統合**: スターリワードやe-Ticketを「お得に支払うための選択肢」と再定義し、`[Pay]`ボタン配下へ統合。情報の分散を防ぎ、ユーザーの認知負荷を軽減。
* **分離**: レジでの「対面決済（画面上部）」と、Web上の「モバイルオーダー（画面下部・中央）」の操作エリアを物理的に分離。ユーザーの脳内モードの切り替えを視覚的にサポートします。

#### Bottom Navの司令塔化（女神ロゴの主動線）
バーガーキング等の先行事例で見られる**「最優先タスクをナビゲーション中央に集約する」**設計思想を分析し、最も頻繁に行う「モバイルオーダー」を中央（女神ロゴ）に配置しました。画面内の情報密度を抑えつつ、アプリを開いた瞬間から迷いなく注文プロセスへ進める「最短の動線」を構築しています。

### 1/15 - 1/19: 商品詳細ページの構造設計
* **作業内容**: 『商品詳細ページ』のHTML実装（5日間）

#### 課題の再定義 (Why)
公式サイトは「モバイルオーダー（非対面）」に特化しているが、実店舗のレジで画面を見せながら注文・決済する**「対面ニーズ」における導線不足**に着目。

#### ソリューション (What)
* **デュアル・アクションバー**: モバイルオーダーと「レジ決済」を並列配置。
* **コンテキスト保持**: 長尺ページでも商品名と価格を見失わないよう、上部固定（`position: sticky`）を導入。

#### 設計のこだわり (UX Design)
「非対面」と「対面」を1画面で完結させる**オムニチャネル対応**。スクロール状況に関わらず「認知（何を選ぶか）」と「実行（どう支払うか）」を親指ゾーンに集約し、操作の迷いを排除。

### 1/12〜14: プロジェクトの再定義（Pivoting）
* **開発の転換点**: 開発を進める中で、実装環境（HTML/CSSのみ）とネイティブアプリの機能差という現実に直面。「自分が何を作るべきか」を深く再考し、以下の通りプロジェクトを再定義した。

* **方針転換**: ネイティブアプリの完全再現を目指すのではなく、Web技術を用いて「アプリ級の操作感」をどこまで実現できるかを検証する。
* **位置付け**: 最終成果物をサービスそのものではなく、**UXの有効性を証明するための「プロトタイプ（PoC）」**として再定義。
* **意義**: 技術的な制約を逆手に取り、Webブラウザという環境下で決済体験を最大化するハックに集中する。

* **商品詳細HTML実装開始**:
    * **情報の構造化**: `product-info > price-group` 等、JS操作を見据えた意味のある階層化を構築。
    * **ボックスモデル制御**: ユニバーサルセレクタによるリセットと `padding` を活用したiPhone幅（393px）基準の精密なレイアウト。
